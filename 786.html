UFCD 786

Instalação e Configuração de Sistemas de Gestão de Bases de Dados

Formador: Ricardo Páscoa

Objetivos

Reconhecer as áreas de intervenção de um administrador de base de dados.

Definir e implementar um plano de instalação de um sistema de base de dados.

Proceder à gestão e instalação de base de dados.

Reflexão

No início desta UFCD foi nos proposto escolher um tema para criarmos uma base de dados de raiz. E logo aqui começou o desafio. Descobrir um programa novo e criar um pequeno projeto.

A minha base de dados consistiu na gestão de uma loja online de óculos de sol e o programa em que desenvolvemos o projeto foi o Microsoft Access.

O elemento base de uma base de dados é a tabela e quando a base de dados ganha dimensões maiores, ou seja, ela torna-se um conjunto de tabelas, então aqui essas tabelas devem relacionar-se, pois são as relações que relacionam os dados das tabelas entre si.

O objetivo de uma base de dados é a gestão dados. Inserir, selecionar, atualizar e eliminar dados, fazer consultas personalizadas, criar formulários personalizados que permitam editar, visualizar e introduzir dados das tabelas, gerar relatórios que possibilitem fazer um resumo e a impressão dos dados organizados.

Na criação de uma tabela devemos seguir algumas regras, pois são estas regras que depois vão determinar se a base de dados está bem construída. Para isso, devemos definir uma chave primária e devemos ter em atenção que pode existir uma tabela central que poderá não ter chave primária e mesmo assim, relaciona-se com as outras tabelas e determina o fluxo de dados na base de dados. Para além da chave primária, é necessário definir o tipo de dados de cada campo, para tornar a base de dados mais eficiente e reduzir o número de erros durante a introdução de dados.

Vimos que a introdução de dados deve ser feita depois da criação das tabelas e estabelecidas as relações.

Ao fim de criada a estrutura da base de dados e depois de inserirmos alguns dados, explorámos algumas funcionalidades do Access. Entre elas, as consultas, os formulários, os relatórios e as macros.

A funcionalidade que para mim foi mais interessante a nível de desenvolvimento, foram os formulários. Estes podem ser criados do zero ou com a ajuda de um assistente, neles podemos adicionar texto, botões, caixas de listagem, subformulários, imagens, gráficos, entre outros. Pude observar como as tabelas interagem entre si, pois no mesmo formulário podemos interagir com várias tabelas e fazer uma atualização dos dados das tabelas. Por último, vimos como funcionam as macros e implementámos macros em formulários.

A avaliação desta UFCD consistiu em duas vertentes: prática e teórica. Na avaliação prática, cada um de nós apresentou a sua base de dados com tudo o que tinha desenvolvido ao longo das aulas. Na avaliação teórica, realizámos um teste de avaliação.
