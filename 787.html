UFCD 787

Administração de Bases de Dados

Formador: Ricardo Páscoa

Objetivos

Proceder à replicação de base de dados e à execução e gestão de cópias de segurança e reposições.

Executar a gestão de transações e a importação e exportação de dados.

Reflexão

O termo base de dados refere-se a um conjunto estruturado de informação / dados. Uma base de dados é uma coleção de dados formalmente definida, informatizada, partilhável e sujeita a um controlo central. Este controlo central é feito pelo administrador da base de dados.

O administrador da base de dados é responsável pela manutenção, atualização e segurança da mesma.

A segurança deve ser uma preocupação constante, porque as ameaças digitais são cada vez mais sofisticadas. Este conceito é aplicável a todos os aspetos de proteção de dados, redes e sistemas informáticos.

No contexto desta UFCD, definimos o que é um plano de segurança informático, diretamente direcionado para a administração de bases de dados. As medidas de segurança que considero mais importantes são: a proteção dos dados, podendo ser utilizada uma criptografia forte; as políticas de backups (cópias de segurança, para que se for necessário, os dados possam ser restaurados em caso de perda dos dados originais e isto pode acontecer por corrupção de ficheiros ou remoções acidentais); o sistema de controlo de acesso, a atribuição de permissões adequadas a utilizadores e aos grupos; a análise de vulnerabilidades na rede; as políticas de instalação e utilização do sistema de gestão de base de dados e todo o software associado; as políticas de utilização da Internet e do correio eletrónico e estabelecer um processo de sensibilização para a segurança informática aos utilizadores no contexto empresarial associado à utilização de uma base de dados. Para reforçar a última medida de segurança, deixo a seguinte citação: “A fim de tornar a Engenharia Social menos provável de ser explorada, é necessário educar todos numa empresa.”

Durante o módulo, explorámos algumas funcionalidades do Microsoft Access, nomeadamente a exportação e importação de tabelas, entre o Access e o Excel.

Vimos o que é um Dashboard e a sua importância para a simplificação e apresentação de dados vitais para a tomada de decisão numa empresa.

Desenvolvemos em contexto de trabalho de grupo, uma solução de implementação informática para uma empresa fictícia. No meu caso, no grupo onde estive inserida, apresentámos uma proposta para um Leilão. Neste trabalho explorámos novamente, a construção de uma base de dados (tabelas e relações), apresentámos uma estrutura de rede e um plano de segurança informático. Para a base de dados, utilizámos o Access e uma segunda alternativa em MySQL; para a estrutura de rede, apresentámos um diagrama de rede, um diagrama para o servidor e algumas medidas para garantir o controlo de rede; no plano de segurança, baseámo-nos na segurança física, no sistema de gestão de base de dados, nas licitações, na rede e na informática geral.

A avaliação desta UFCD foi composta pela realização do trabalho de grupo e pelo teste escrito.

O maior desafio que senti, foi no contexto de trabalhar em grupo, pois o grupo era constituído por 6 elementos. É sempre difícil coordenar grupos e trabalhar em grupo: organizar ideias, desenvolver o tema central e implementar a melhor estratégia de apresentação. Foi uma experiência nova e que apesar das dificuldades referidas, resultou na melhor resolução possível, dentro das capacidades de cada um.
