UFCD 773

Rede Local - Instalação

Formador: Fernando Tavares

Objetivos

Reconhecer a estrutura de um sistema operativo.

Definir um plano de instalação de um servidor de rede local.

Reflexão

No início desta UFCD começámos por ver as diferenças entre um sistema operativo local (SOL) e um sistema operativo de rede (SOR). O primeiro tem com objetivo gerir os recursos de hardware e o segundo o fornecimento de serviços locais para o cliente remoto. Mais posso acrescentar, que um sistema operativo de rede é um conjunto de módulos específicos que potenciam o sistema operativo local, que é projetado exclusivamente para servidores, permitindo assim que o servidor administre os utilizadores, dados, grupos, segurança, aplicativos e outras funções de rede. Na medida de desenvolvimento dos nossos conhecimentos num sistema operativo de rede foi-nos introduzido o Windows Server 2022, em que o mesmo tem determinados requisitos mínimos de hardware para podermos fazer uma instalação.

Vimos que existem dois tipos de servidores: em rack e em torre. Dando seguimento a este conteúdo, foi visto o que é o hot plugging (a capacidade de substituir ou instalar um dispositivo sem desligar o computador conectado) e que um backup e armazenamento de dados é feito em disco ou em banda.

Em atividade de laboratório fizemos a instalação do Windows Server 2022 criando uma máquina virtual no VirtualBox. Depois da instalação, fizemos algumas configurações, nomeadamente: alterar o fuso horário, alterar a língua, mudar o nome do computador e configurar o endereço de IP manualmente. Instalámos a função Active Directory Domain Services (AD DS), promovemos um controlador de domínio e criámos um utilizador.

Porque um servidor está ligado continuamente, para evitar um corte inesperado de energia, a UPS - fonte de alimentação ininterrupta - fornece energia aos equipamentos sensíveis ligados à mesma.

Foram dadas as definições de função, serviços de função e funcionalidades de um servidor e foram apresentados alguns tipos de servidores de rede, dentro dos quais se destacam: o controlador de domínio (servidor de autenticação), o servidor de DHCP e o servidor de DNS.

Em termos de segurança da informação, o controlo de acesso, é feito através dos processos de autenticação, autorização e auditoria.

Para consolidação dos nossos conhecimentos realizámos um trabalho escrito e um teste de avaliação.

Os meus conhecimentos em servidores no início desta UFCD eram quase nulos mas posso dizer que apesar de ainda sentir algumas dificuldades e dúvidas acerca do funcionamento dos mesmos, hoje sei fazer uma instalação de um servidor e algumas configurações.
