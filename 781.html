UFCD 781

Análise de Sistemas de Informação

Formador: João Valente

Objetivos

Reconhecer e utilizar as diferentes metodologias de análise de sistemas de informação, no âmbito do processo de informatização de uma organização.

Reflexão

Um sistema de informação é um modelo de processos responsáveis por guardar e transmitir dados que sejam úteis ao desenvolvimento de produtos ou serviços das empresas, organizações ou projetos. É do sistema de informação, orientar a tomada de decisão nos três níveis de responsabilidade: operacional, tático e estratégico. Igualmente, o S.I. deve assegurar a segurança e a futura disponibilidade da informação. São componentes de um S.I., os dados, o sistema de processamento de dados e os canais de comunicação.

Estando nós na era da informação e da globalização, temos que ter em conta que quando se armazenam os dados, tem de ser definida a forma como os dados vão ser recuperados e identificados. Os processos de guardar e de obter dados são dos mais importantes no software de hoje em dia. Por isso, para minimizar o tempo de resposta do sistema durante estas operações, é preciso que as formas com que acedemos à memória sejam eficazes. Inicialmente, a única solução era guardar os dados das aplicações em ficheiros. Mas havia uma necessidade de gerir e criar uma independência entre os dados e os programas. Para solucionar estes problemas, por volta dos anos 60, surgiram os primeiros sistemas de gestão de bases de dados (SGBD) - um conjunto de ficheiros organizados.

No desenho de um S.I. moderno, o grupo de componentes a considerar devem constar as bases de dados. O desenho do S.I. produz os detalhes que determinam como o sistema cumprirá as especificações identificadas durantes as atividades iniciais do ciclo de desenvolvimento.

Nesta UFCD abordámos tudo o que foi mencionado anteriormente, desde a evolução do processamento de dados, as fases de criação de uma base de dados, o que é uma base de dados, um sistema de base de dados e um sistema de gestão de bases de dados. Percebemos quais são as componentes de um sistema de gestão de bases de dados e a sua arquitetura.

Vimos o ciclo de desenvolvimento do software e os diagramas que para mim, mais se destacaram foram: o diagrama de fluxo de dados e o diagrama Entidade-Relação (modelo E-R). O primeiro, mostra procedimentos e fluxo de dados entre processos, pois trata-se de uma ferramenta de projeto de alto nível para visualizar os procedimentos necessários; o segundo, é um diagrama que mostra o percurso de acesso num modelo de dados ou base de dados (estrutura), que é usado por um processo ou procedimento, com uso frequente no projeto de bases de dados.

Desenvolvemos com exercícios o modelo E-R e o modelo relacional e estabelecemos as diferenças entre eles. Analisámos as etapas da normalização de dados e na descrição detalhada de dados, definimos o que é uma chave primária, uma chave secundária e uma chave estrangeira.

Vimos ainda o que são entidades (as tabelas de uma possível base de dados), atributos (os dados que contêm informação que descrevem uma entidade) e as relações (a sua cardinalidade).

Esta UFCD é muito importante para complementar o desenvolvimento em bases de dados, desde a representação dos processos, o desenho lógico e a normalização das tabelas.
